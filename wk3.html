Week 3 Quiz: React Navigation Revisited

1. What is the difference between dynamic and static navigation in React Navigation v7?

  • A) Dynamic navigation requires more code and is instantiated as a variable reference
  • B) Static navigation requires creating a separate configuration file
  • C) Dynamic navigation is no longer supported in React Navigation v7
  • D) Static navigation must always be used for deep linking
Answer: A) Dynamic navigation requires more code and is instantiated as a variable reference

2. Which of the following is an advantage of using static navigation in React Navigation v7?

  • A) It eliminates the need for a NavigationContainer
  • B) It integrates better with TypeScript by defining static types
  • C) It does not require any configuration
  • D) It automatically generates deep links
Answer: B) It integrates better with TypeScript by defining static types

3. Which React Navigation method is used to define deep linking in static navigation?

  • A) screenOptions
  • B) navigation.navigate()
  • C) createStaticNavigation()
  • D) screen param in the route object
Answer: D) screen param in the route object

4. Which navigator in React Navigation provides a way to mimic native OS navigation patterns?

  • A) BottomTabs
  • B) Stack
  • C) Drawer
  • D) MaterialTopTabs
Answer: B) Stack

5. What is the main difference between Stack and NativeStack navigators?

  • A) NativeStack is faster and optimized for performance using OS-specific APIs
  • B) Stack navigator is deprecated in React Navigation v7
  • C) NativeStack does not support navigation parameters
  • D) Stack navigator cannot handle deep linking
Answer: A) NativeStack is faster and optimized for performance using OS-specific APIs

6. Which navigator in React Navigation is commonly used for tab-based navigation?

  • A) Drawer
  • B) Stack
  • C) BottomTabs
  • D) NativeStack
Answer: C) BottomTabs

7. Which React Navigation method allows for navigating between nested navigators?

  • A) jumpTo()
  • B) replace()
  • C) navigate()
  • D) openDrawer()
Answer: C) navigate()

8. In React Navigation, which hook is used to retrieve the navigation object?

  • A) useRoute
  • B) useState
  • C) useNavigation
  • D) useEffect
Answer: C) useNavigation

9. Which navigation method is specifically used for tab navigation?

  • A) navigate()
  • B) jumpTo()
  • C) push()
  • D) popToTop()
Answer: B) jumpTo()

10. What is the purpose of the getParent() method in React Navigation?

  • A) To retrieve the state of a parent navigator
  • B) To navigate to a nested screen
  • C) To clear navigation history
  • D) To close a modal
Answer: A) To retrieve the state of a parent navigator

11. What is the purpose of the push() method in React Navigation?

  • A) To navigate to a new screen and add it to the stack
  • B) To go back to the previous screen
  • C) To replace the current screen
  • D) To remove all screens from the stack
Answer: A) To navigate to a new screen and add it to the stack

12. What does the goBack() method do in React Navigation?

  • A) Navigates to a specific screen
  • B) Goes back to the previous screen
  • C) Reloads the current screen
  • D) Removes all screens from the stack
Answer: B) Goes back to the previous screen

13. What parameter is required when passing data between screens in React Navigation?

  • A) route.params
  • B) screenProps
  • C) navigatorConfig
  • D) stackOptions
Answer: A) route.params

14. What is the primary use of the replace() method in React Navigation?

  • A) To push a new screen onto the stack
  • B) To replace the current screen with a new one
  • C) To close the application
  • D) To reset the navigation state
Answer: B) To replace the current screen with a new one

15. What is the purpose of the reset() method in React Navigation?

  • A) To clear the navigation history and set a new route stack
  • B) To reload the current screen
  • C) To go back to the home screen
  • D) To update the current route params
Answer: A) To clear the navigation history and set a new route stack

16. What does the popToTop() method do in React Navigation?

  • A) Removes all screens except the first one
  • B) Replaces the current screen with the home screen
  • C) Goes back one screen
  • D) Prevents the user from navigating
Answer: A) Removes all screens except the first one

17. What is the purpose of the addListener() method in React Navigation?

  • A) To listen for navigation events
  • B) To add a new screen dynamically
  • C) To reset navigation parameters
  • D) To force a screen re-render
Answer: A) To listen for navigation events

18. What is the main function of the Drawer Navigator in React Navigation?

  • A) To create a sidebar navigation menu
  • B) To manage deep linking
  • C) To replace the Stack Navigator
  • D) To update screen parameters
Answer: A) To create a sidebar navigation menu

19. What is the default transition style used in Stack Navigator?

  • A) Slide from the right
  • B) Fade
  • C) Scale
  • D) None
Answer: A) Slide from the right

20. Which React Navigation component allows for bottom tab navigation?

  • A) createBottomTabNavigator
  • B) createStackNavigator
  • C) createDrawerNavigator
  • D) createSwitchNavigator
Answer: A) createBottomTabNavigator

21. What is the purpose of the gestureEnabled option in Stack Navigator?

  • A) Enables or disables gesture-based navigation
  • B) Forces a transition animation
  • C) Prevents navigation between screens
  • D) Disables deep linking
Answer: A) Enables or disables gesture-based navigation

22. How do you specify custom transition animations in React Navigation?

  • A) Using the transitionConfig prop
  • B) By modifying the stackOptions
  • C) With a separate transition handler
  • D) By overriding the navigate() method
Answer: B) By modifying the stackOptions

23. What is the default behavior of the back button in Stack Navigator?

  • A) It exits the application
  • B) It navigates to the home screen
  • C) It goes back to the previous screen
  • D) It refreshes the current screen
Answer: C) It goes back to the previous screen

24. What is the purpose of initialRouteName in a navigator configuration?

  • A) Defines the default screen when the navigator loads
  • B) Sets the route transition style
  • C) Specifies the parent navigator
  • D) Forces a screen refresh
Answer: A) Defines the default screen when the navigator loads

25. What is the primary use case for the pop() method in Stack Navigator?

  • A) To remove the top screen from the stack
  • B) To navigate to a new screen
  • C) To replace the current screen
  • D) To restart the application
Answer: A) To remove the top screen from the stack

26. What is the purpose of the headerShown property in screen options?

  • A) Determines whether the header is visible
  • B) Specifies the transition animation
  • C) Forces a screen re-render
  • D) Disables navigation gestures
Answer: A) Determines whether the header is visible

27. What does the screenOptions property define in a navigator?

  • A) Global styling and behavior for all screens
  • B) The default route for deep linking
  • C) The route transition style
  • D) A static configuration for route parameters
Answer: A) Global styling and behavior for all screens

28. What is the primary advantage of using useNavigation() hook?

  • A) Provides access to the navigation object without prop drilling
  • B) Forces a screen re-render
  • C) Prevents navigation events from being triggered
  • D) Allows defining navigation outside the component tree
Answer: A) Provides access to the navigation object without prop drilling

29. How can you programmatically set parameters for a screen in React Navigation?

  • A) Using navigation.setParams()
  • B) By modifying the route object directly
  • C) Through useState() hook
  • D) With a global configuration file
Answer: A) Using navigation.setParams()

30. What is the recommended way to implement deep linking in React Navigation?

  • A) Define linking configuration in NavigationContainer
  • B) Manually parse URL parameters
  • C) Use localStorage to save navigation state
  • D) Create a separate navigator instance
Answer: A) Define linking configuration in NavigationContainer
