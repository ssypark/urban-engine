Week 4 Quiz: Data Persistence

1. What is data persistence in React Native?

  • A) The ability to store data temporarily in the app's memory
  • B) The process of retrieving data from an API
  • C) The ability to store data even after the app is closed and reopened
  • D) A method to optimize animations in React Native
Answer: C) The ability to store data even after the app is closed and reopened

2. What is one common approach to storing persistent data in React Native?

  • A) Using Redux for state management
  • B) Using AsyncStorage
  • C) Using an API call for every data request
  • D) Using session storage
Answer: B) Using AsyncStorage

3. What is a key limitation of app-specific internal storage?

  • A) It cannot store key-value pairs
  • B) It requires an internet connection
  • C) It is accessible only by the app that wrote it
  • D) It automatically syncs across devices
Answer: C) It is accessible only by the app that wrote it

4. What data structure does AsyncStorage use to store information?

  • A) JSON-formatted key-value pairs
  • B) Relational database tables
  • C) XML documents
  • D) Plaintext files
Answer: A) JSON-formatted key-value pairs

5. Why is AsyncStorage considered "unsecure" for sensitive data?

  • A) It requires an active internet connection
  • B) It can be accessed by any installed app
  • C) It is not encrypted by default
  • D) It has a limited storage capacity
Answer: C) It is not encrypted by default

6. What is the correct way to install AsyncStorage?

  • A) npm install async-storage
  • B) npm install @react-native-async-storage/async-storage
  • C) npm install react-native-storage
  • D) npm install redux-persist
Answer: B) npm install @react-native-async-storage/async-storage

7. Which method is used to store data in AsyncStorage?

  • A) setItem()
  • B) storeData()
  • C) pushItem()
  • D) saveData()
Answer: A) setItem()

8. Which AsyncStorage method retrieves stored data?

  • A) getData()
  • B) fetchItem()
  • C) getItem()
  • D) retrieveValue()
Answer: C) getItem()

9. What is the purpose of try/catch blocks when working with AsyncStorage?

  • A) To prevent app crashes due to storage errors
  • B) To speed up data retrieval
  • C) To encrypt data automatically
  • D) To optimize app performance
Answer: A) To prevent app crashes due to storage errors

10. What happens when the removeItem() method is used in AsyncStorage?

  • A) It deletes all stored data
  • B) It removes a specific key-value pair
  • C) It encrypts the stored data
  • D) It syncs the data with the server
Answer: B) It removes a specific key-value pair

11. What is a major benefit of using SecureStore over AsyncStorage?

  • A) It is faster
  • B) It encrypts stored data
  • C) It allows unlimited storage
  • D) It requires no installation
Answer: B) It encrypts stored data

12. Which storage option is best suited for storing sensitive user credentials?

  • A) AsyncStorage
  • B) SecureStore
  • C) LocalStorage
  • D) SessionStorage
Answer: B) SecureStore

13. What method does SecureStore provide for storing data?

  • A) setItem()
  • B) storeAsync()
  • C) setSecureItem()
  • D) saveData()
Answer: A) setItem()

14. What is the key advantage of using SQLite in a React Native app?

  • A) It provides structured relational storage
  • B) It is faster than using Redux
  • C) It stores data in the cloud automatically
  • D) It does not require any installation
Answer: A) It provides structured relational storage

15. What type of data is best suited for Firebase Firestore?

  • A) Large files like images
  • B) Highly structured relational data
  • C) JSON-based real-time data
  • D) Local-only storage
Answer: C) JSON-based real-time data

16. How does Redux Persist help with data persistence?

  • A) It stores Redux state in AsyncStorage or local storage
  • B) It provides automatic database backups
  • C) It prevents state from changing
  • D) It encrypts all stored data
Answer: A) It stores Redux state in AsyncStorage or local storage

17. What is a downside of using Firebase Firestore for offline-first applications?

  • A) It does not support offline mode
  • B) Data sync may cause conflicts when reconnecting
  • C) It requires complex SQL queries
  • D) It lacks cloud-based storage
Answer: B) Data sync may cause conflicts when reconnecting

18. Which of the following methods is NOT used for data persistence?

  • A) AsyncStorage
  • B) SecureStore
  • C) Redux Reducers
  • D) SQLite
Answer: C) Redux Reducers

19. Why is Firebase often chosen for mobile applications?

  • A) It provides real-time database sync
  • B) It requires no authentication
  • C) It is built into React Native
  • D) It only supports offline storage
Answer: A) It provides real-time database sync

20. What command is used to install Firebase in a React Native project?

  • A) npm install firebase
  • B) npm install @react-native-firebase/app
  • C) npm install redux-firebase
  • D) npm install firestore-react
Answer: B) npm install @react-native-firebase/app

21. What is the primary reason for using an indexed database in React Native?

  • A) It reduces app bundle size
  • B) It improves performance for structured queries
  • C) It prevents data loss during updates
  • D) It allows cross-platform compatibility
Answer: B) It improves performance for structured queries

22. How does WatermelonDB optimize database operations?

  • A) By storing data in a compressed format
  • B) By offloading processing to native threads
  • C) By caching all responses in Redux
  • D) By using GraphQL for data retrieval
Answer: B) By offloading processing to native threads

23. What is the main advantage of using Realm over AsyncStorage?

  • A) It has built-in encryption
  • B) It does not require an internet connection
  • C) It automatically syncs data to Firebase
  • D) It is easier to debug
Answer: A) It has built-in encryption

24. Which database is optimized for handling large datasets in mobile applications?

  • A) AsyncStorage
  • B) SQLite
  • C) SecureStore
  • D) LocalStorage
Answer: B) SQLite

25. What is a key drawback of using Redux Persist?

  • A) It requires a database connection
  • B) It may slow down app startup time
  • C) It only works online
  • D) It cannot store JSON data
Answer: B) It may slow down app startup time

26. What storage solution is best for storing large amounts of relational data locally?

  • A) SecureStore
  • B) Firebase Firestore
  • C) SQLite
  • D) AsyncStorage
Answer: C) SQLite

27. How does WatermelonDB handle offline data synchronization?

  • A) It syncs automatically with Firebase
  • B) It uses a background process for sync
  • C) It requires manual data entry
  • D) It does not support offline sync
Answer: B) It uses a background process for sync

28. What makes Firebase a strong choice for real-time applications?

  • A) It uses NoSQL structure with automatic updates
  • B) It has built-in SQL support
  • C) It allows unlimited data storage
  • D) It is only available for enterprise users
Answer: A) It uses NoSQL structure with automatic updates

29. What is the purpose of setItem() in AsyncStorage?

  • A) It removes an item from storage
  • B) It stores a key-value pair
  • C) It fetches data from storage
  • D) It encrypts data before storing
Answer: B) It stores a key-value pair

30. How can developers ensure secure data storage in React Native?

  • A) By using SecureStore for sensitive data
  • B) By always storing data in AsyncStorage
  • C) By disabling local caching
  • D) By removing all stored data after every session
Answer: A) By using SecureStore for sensitive data
