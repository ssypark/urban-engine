Week 2 Quiz: Promises and Data Binding

1. What is the key difference between synchronous and asynchronous requests?

  • A) Synchronous requests execute all at once, while asynchronous requests wait for execution.
  • B) Synchronous requests block further execution until a response is received, while asynchronous requests allow continued execution.
  • C) Asynchronous requests reload the entire page, while synchronous requests do not.
  • D) There is no difference between synchronous and asynchronous requests.
Answer: B) Synchronous requests block further execution until a response is received, while asynchronous requests allow continued execution.

2. What was one of the first web technologies that introduced asynchronous functionality?

  • A) Java Applets
  • B) AJAX
  • C) Fetch API
  • D) WebSockets
Answer: A) Java Applets

3. Why did Java Applets lose popularity despite providing asynchronous functionality?

  • A) They were difficult to develop.
  • B) AJAX provided a simpler alternative.
  • C) Browser support for Java Applets declined.
  • D) All of the above.
Answer: D) All of the above.

4. What is the main advantage of asynchronous requests?

  • A) They allow the client to make multiple requests without waiting for each response.
  • B) They ensure requests return in the order they were sent.
  • C) They prevent page updates.
  • D) They block execution until a response is received.
Answer: A) They allow the client to make multiple requests without waiting for each response.

5. What are the three possible states of a JavaScript Promise?

  • A) Pending, Fulfilled, Rejected
  • B) Start, Processing, End
  • C) Request, Response, Completed
  • D) Created, Sent, Received
Answer: A) Pending, Fulfilled, Rejected

6. What JavaScript method is used to handle a resolved Promise?

  • A) .resolve()
  • B) .catch()
  • C) .then()
  • D) .finally()
Answer: C) .then()

7. What method is used in JavaScript to handle errors in a Promise chain?

  • A) .catch()
  • B) .error()
  • C) .reject()
  • D) .finally()
Answer: A) .catch()

8. What is the purpose of chaining Promises?

  • A) To handle multiple asynchronous operations in sequence.
  • B) To avoid executing functions sequentially.
  • C) To prevent API calls from failing.
  • D) To execute all JavaScript code synchronously.
Answer: A) To handle multiple asynchronous operations in sequence.

9. What problem does Promise.all() solve?

  • A) It ensures multiple Promises resolve at the same time.
  • B) It forces Promises to resolve sequentially.
  • C) It prevents API errors.
  • D) It blocks code execution until all Promises fail.
Answer: A) It ensures multiple Promises resolve at the same time.

10. What is the advantage of using .finally() in a Promise chain?

  • A) It runs whether the Promise is resolved or rejected.
  • B) It prevents API errors.
  • C) It ensures all Promises run in order.
  • D) It cancels all other Promises.
Answer: A) It runs whether the Promise is resolved or rejected.

11. What is data binding in JavaScript frameworks?

  • A) The process of linking data between the UI and business logic.
  • B) A method for making API requests.
  • C) A security feature for encrypting API data.
  • D) A function that allows multiple HTTP requests.
Answer: A) The process of linking data between the UI and business logic.

12. What are the two types of data binding?

  • A) One-way and two-way binding.
  • B) Client-side and server-side binding.
  • C) Static and dynamic binding.
  • D) Direct and indirect binding.
Answer: A) One-way and two-way binding.

13. Which type of data binding allows real-time updates between the model and the view?

  • A) One-way binding.
  • B) Two-way binding.
  • C) Static binding.
  • D) Manual binding.
Answer: B) Two-way binding.

14. Which JavaScript framework is known for using two-way data binding?

  • A) React
  • B) Vue.js
  • C) Angular
  • D) jQuery
Answer: C) Angular

15. In React, how is one-way data binding typically achieved?

  • A) Using state and props.
  • B) Using inline JavaScript.
  • C) By modifying the DOM directly.
  • D) Using jQuery methods.
Answer: A) Using state and props.

16. What is the main purpose of event listeners in JavaScript?

  • A) To directly modify the DOM
  • B) To handle user interactions asynchronously
  • C) To block page execution
  • D) To reload the page on every event
Answer: B) To handle user interactions asynchronously

17. Which of the following best describes the role of the Virtual DOM in React?

  • A) A backup copy of the actual DOM
  • B) A programming language for APIs
  • C) A lightweight copy of the real DOM used to optimize updates
  • D) A method to remove unnecessary UI components
Answer: C) A lightweight copy of the real DOM used to optimize updates

18. In JavaScript, what does the async keyword do when used in a function?

  • A) Makes the function return a Promise
  • B) Automatically executes the function
  • C) Prevents the function from running
  • D) Blocks the execution of all subsequent code
Answer: A) Makes the function return a Promise

19. What is the correct way to handle an async function error?

  • A) Using try...catch
  • B) Using console.log()
  • C) Using an alert
  • D) Ignoring the error
Answer: A) Using try...catch

20. Which lifecycle method is used in React to fetch data after a component mounts?

  • A) componentDidMount
  • B) componentWillUnmount
  • C) render()
  • D) useEffect
Answer: D) useEffect

21. What is the primary benefit of using the useState hook in React?

  • A) It allows functional components to manage state
  • B) It replaces the need for Redux
  • C) It eliminates the need for rendering components
  • D) It makes components render synchronously
Answer: A) It allows functional components to manage state

22. What does the useEffect hook do in React?

  • A) Executes side effects after a component renders
  • B) Handles UI styling
  • C) Forces a component to update
  • D) Prevents API calls from executing
Answer: A) Executes side effects after a component renders

23. What is the default behavior of useEffect in React if no dependency array is provided?

  • A) It runs only once after the initial render
  • B) It runs on every render
  • C) It does not run at all
  • D) It runs only when state updates
Answer: B) It runs on every render

24. What does the dependency array in useEffect do?

  • A) Prevents the effect from running
  • B) Specifies when the effect should re-run
  • C) Forces an immediate component re-render
  • D) Removes the effect from memory
Answer: B) Specifies when the effect should re-run

25. What does returning a function inside useEffect achieve?

  • A) It prevents memory leaks by cleaning up effects
  • B) It stops the component from re-rendering
  • C) It forces synchronous execution
  • D) It disables event listeners
Answer: A) It prevents memory leaks by cleaning up effects

26. What is the primary advantage of using async/await over traditional promise chaining?

  • A) It improves readability and maintainability
  • B) It makes API calls execute faster
  • C) It prevents all errors in JavaScript
  • D) It automatically retries failed requests
Answer: A) It improves readability and maintainability

27. What is the purpose of the fetch() function in JavaScript?

  • A) To send HTTP requests and retrieve responses
  • B) To update the DOM without reloading
  • C) To handle form validation
  • D) To bind data to UI components
Answer: A) To send HTTP requests and retrieve responses

28. What happens if an API request fails inside an async function without a try/catch block?

  • A) The function crashes the entire application
  • B) The promise is rejected, but the error is not handled
  • C) The request automatically retries
  • D) The function returns null
Answer: B) The promise is rejected, but the error is not handled

29. What is the purpose of the .json() method when using fetch()?

  • A) To parse the response body as JSON
  • B) To stringify an object before sending it
  • C) To encrypt API responses
  • D) To close the API connection
Answer: A) To parse the response body as JSON

30. What does the term "side effect" refer to in React?

  • A) Any operation that affects the outside world
  • B) A bug in the application
  • C) A built-in function in React
  • D) A redundant component update
Answer: A) Any operation that affects the outside world
