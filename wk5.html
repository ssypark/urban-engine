Week 5 Quiz: Context and Global State

1. What is the primary way state variables are established in React applications?

  • A) Using the setState() method in functional components
  • B) Using the useState() hook for functional components
  • C) Using Redux for state management
  • D) Using global variables
Answer: B) Using the useState() hook for functional components

2. What happens when a state variable updates in React?

  • A) The entire application reloads
  • B) The component where the state is used re-renders
  • C) Only the parent component re-renders
  • D) Nothing happens
Answer: B) The component where the state is used re-renders

3. Why can managing state in complex applications become challenging?

  • A) Components cannot share state variables
  • B) State variables cannot persist across component unmounts
  • C) Multiple components may need access to the same state
  • D) React does not support state management
Answer: C) Multiple components may need access to the same state

4. What is a common example of an application-wide state?

  • A) UI button color
  • B) Login status
  • C) Page title
  • D) CSS styles
Answer: B) Login status

5. How does React typically handle state lifting?

  • A) By making state variables global
  • B) By passing state down through props
  • C) By using Redux for every application
  • D) By storing state in localStorage
Answer: B) By passing state down through props

6. What is the downside of prop drilling?

  • A) It increases performance
  • B) It makes debugging easier
  • C) It makes component reusability harder
  • D) It is the best way to manage state
Answer: C) It makes component reusability harder

7. How does React Context solve prop drilling?

  • A) By allowing state to be accessed globally
  • B) By forcing all components to re-render
  • C) By replacing all prop usage
  • D) By storing state in an external database
Answer: A) By allowing state to be accessed globally

8. What function is used to create a new React Context?

  • A) createGlobalState()
  • B) createContext()
  • C) useReducer()
  • D) useContext()
Answer: B) createContext()

9. What hook is used in a component to access a React Context?

  • A) useEffect()
  • B) useState()
  • C) useContext()
  • D) useReducer()
Answer: C) useContext()

10. What is a key disadvantage of using React Context for state management?

  • A) It requires external libraries
  • B) It does not work with functional components
  • C) It may cause unnecessary re-renders
  • D) It is harder to debug than prop drilling
Answer: C) It may cause unnecessary re-renders

11. How do you provide a React Context to multiple components?

  • A) By wrapping components in a Context.Provider
  • B) By passing context as a prop
  • C) By using Redux
  • D) By calling useContext() in every component
Answer: A) By wrapping components in a Context.Provider

12. What does the Context.Provider component do?

  • A) It creates a new global state
  • B) It provides the context value to all child components
  • C) It automatically updates the componentâ€™s props
  • D) It replaces useState()
Answer: B) It provides the context value to all child components

13. What problem does useReducer solve when used with React Context?

  • A) It prevents unnecessary re-renders
  • B) It removes the need for state
  • C) It replaces useContext()
  • D) It stores data in the cloud
Answer: A) It prevents unnecessary re-renders

14. What is the purpose of useReducer in state management?

  • A) To create multiple independent contexts
  • B) To manage complex state logic more effectively
  • C) To replace all useState hooks
  • D) To automatically sync state with an API
Answer: B) To manage complex state logic more effectively

15. What does a reducer function return in React?

  • A) A new state value based on the previous state
  • B) A prop value
  • C) A new React component
  • D) A UI change
Answer: A) A new state value based on the previous state

16. What parameter is passed into a reducer function?

  • A) The previous state and an action
  • B) The entire React tree
  • C) A context object
  • D) A function reference
Answer: A) The previous state and an action

17. How does useReducer differ from useState?

  • A) useReducer is only used in class components
  • B) useReducer is better for managing complex state logic
  • C) useReducer does not update state
  • D) useReducer does not require an initial state
Answer: B) useReducer is better for managing complex state logic

18. What is the purpose of the dispatch function in useReducer?

  • A) It updates the React tree
  • B) It triggers the reducer function to update state
  • C) It replaces useContext()
  • D) It stores state in Redux
Answer: B) It triggers the reducer function to update state

19. What advantage does Redux have over React Context?

  • A) Redux provides better debugging tools
  • B) Redux is built into React
  • C) Redux automatically updates all components
  • D) Redux does not require reducers
Answer: A) Redux provides better debugging tools

20. What is a downside of using Redux?

  • A) It requires additional boilerplate code
  • B) It does not work with functional components
  • C) It does not support middleware
  • D) It automatically persists data
Answer: A) It requires additional boilerplate code

21. What is the main reason developers use Redux over Context API?

  • A) Redux requires less code
  • B) Redux has built-in persistence
  • C) Redux provides better state predictability and debugging
  • D) Redux is faster than Context API
Answer: C) Redux provides better state predictability and debugging

22. Which React function allows components to subscribe to global state values in Context API?

  • A) useState()
  • B) useReducer()
  • C) useContext()
  • D) useEffect()
Answer: C) useContext()

23. What is the recommended way to structure a large-scale React application using global state?

  • A) Use one large Context for everything
  • B) Use multiple smaller Contexts for different features
  • C) Store all state in localStorage
  • D) Avoid using state
Answer: B) Use multiple smaller Contexts for different features

24. What should be avoided when using React Context for state management?

  • A) Nesting Context Providers deeply
  • B) Using multiple Contexts
  • C) Using default values
  • D) Using reducers for state changes
Answer: A) Nesting Context Providers deeply

25. Why might you use middleware with Redux?

  • A) To enhance debugging and logging
  • B) To replace reducers
  • C) To store UI components
  • D) To remove the need for action types
Answer: A) To enhance debugging and logging

26. What problem does Redux Toolkit aim to solve?

  • A) It simplifies Redux state management by reducing boilerplate
  • B) It removes the need for React
  • C) It eliminates the need for reducers
  • D) It is only used for large applications
Answer: A) It simplifies Redux state management by reducing boilerplate

27. What is a potential downside of using Redux for state management?

  • A) Redux cannot be used with React
  • B) Redux requires additional boilerplate code
  • C) Redux does not support middleware
  • D) Redux automatically syncs with databases
Answer: B) Redux requires additional boilerplate code

28. How does Redux improve debugging?

  • A) It stores all previous state changes
  • B) It prevents state changes
  • C) It logs errors automatically
  • D) It replaces React DevTools
Answer: A) It stores all previous state changes

29. When should you choose Redux over Context API?

  • A) When managing simple UI state
  • B) When managing complex global state across many components
  • C) When working only with small applications
  • D) When replacing CSS
Answer: B) When managing complex global state across many components

30. What best describes the difference between Redux and React Context?

  • A) Redux is optimized for large-scale state management, while Context is best for simple state sharing
  • B) Redux is built into React
  • C) Context does not support state updates
  • D) Redux replaces the need for all local component states
Answer: A) Redux is optimized for large-scale state management, while Context is best for simple state sharing
